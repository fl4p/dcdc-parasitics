# Package Lead Inductance in TO-220/TO-247-Class Power Devices

## Abstract

Through-hole power packages add nanohenries of inductance in exactly the
current paths that dominate fast-switching converter behavior. The important
quantity is not only total commutation-loop inductance. Common-source
inductance (CSI), the part shared by the power path and gate-drive reference,
feeds `L * di/dt` back into `Vgs`, slows transitions, increases switching loss,
and can excite gate-loop ringing or false turn-on.

The literature supports three practical conclusions:

1. TO-220 and TO-247 package source inductance is commonly several nH before
   any board copper is counted.
2. Package inductance is topology-specific. A useful model must preserve drain,
   source, gate, internal interconnect, and mutual terms when they share
   switching or drive loops.
3. Ringing-derived loop inductance is useful, but only after the capacitance,
   current path, damping, fixture inductance, and any package inductance already
   present in the device SPICE model are accounted for.

## Definitions

`Package lead inductance` is the inductance of the conductive path inside and
immediately at the package: external leads, lead frame, die attach path, clips,
bond wires, and internal die-to-die links. It is not the same thing as board
copper inductance.

`Insertion inductance` is the inductance a package contributes to the main
commutation loop. For a diode or two-terminal current path this is the
forward-current loop through one lead, bond wires or clip, die, tab/lead frame,
and the return lead or tab.

`Common-source inductance` is the source-side inductance shared by the main
drain-source current and the gate-driver reference current. It matters more than
its nH value suggests because it directly modulates effective gate-source
voltage:

```text
Vgs_eff = Vgate - Vsource - Lcsi * dIds/dt
```

For a cascode or multi-die device, "source inductance" is not a single obvious
element. The critical CSI terms are the interconnects shared by the internal
power loop and the internal gate-control loops.

## Measured Package-Scale Values

Aikawa et al. measured CSI directly on devices mounted to PCBs using only a
signal generator and oscilloscope. Their method turns the device on, injects an
AC drain-source current, and measures the induced source/gate-reference voltage.
The PCB fixture is part of the measurement: the drain-source injection path and
gate-source sensing path must share no PCB current path except at the source
terminal, avoid overlap, and use a ground plane. Otherwise fixture mutual
inductance appears as extra measured CSI. With measured voltage amplitude
`Vac`, injected current `Iac`, phase `phi`, and angular frequency `omega`, the
source inductance is:

```text
Ls = Vac * sin(phi) / (omega * Iac)
```

Their reported CSI database is a useful first-order prior:

| Package | Measured CSI range |
| --- | ---: |
| TO-247 | 5.5 to 7.8 nH |
| TO-220 | 3.9 to 5.7 nH |
| I-PAK | 3.7 to 4.3 nH |
| D2-PAK | 4.9 to 6.1 nH |
| D-PAK | 2.3 to 3.2 nH |
| SOT-223 | 2.3 to 2.6 nH |
| SOT-23 | 1.4 to 1.6 nH |

The important detail is the spread. In TO-220 and TO-247, roughly one third of
the total CSI varied across devices of the same package. After opening packages,
Aikawa et al. attributed that spread mainly to the source bond wires: higher
current devices tended to use more and/or thicker parallel wires, reducing the
bond-wire contribution.

For a rough bond-wire estimate, they used Grover's round-wire formula. For `N`
parallel wires of length `l` and diameter `r`:

```text
Lbw ~= (mu0 * l / (2 * pi * N)) * (ln(4 * l / r) - 3/4)
```

This is not a full package model, but it explains why two nominal TO-220 devices
can differ by more than 1 nH without any PCB change.

## Multi-Loop Package Models

Liu et al. studied a 600 V cascode GaN HEMT in a TO-220 package and extracted
package parasitics with Ansoft Q3D. The paper is valuable because it does not
collapse the package into a single loop inductor. It defines turn-on and
turn-off current directions, includes mutual coupling sign, and then verifies
the model with double-pulse testing.

Extracted TO-220 cascode package terms were:

| Element | Turn-on | Turn-off |
| --- | ---: | ---: |
| Internal link `Lint1` | 0.27 nH | 0.26 nH |
| Internal link `Lint2` | 0.23 nH | 0.17 nH |
| Internal link `Lint3` | 0.24 nH | 0.43 nH |
| Source lead `Ls` | 0.57 nH | 0.89 nH |
| Gate lead `Lg` | 2.87 nH | 3.12 nH |
| Drain lead `Ld` | 1.89 nH | 1.62 nH |

The absolute internal-link values are small, but their loop placement makes them
critical. Removing `Ls`, `Lint1`, or `Lint3` in simulation reduced 400 V / 10 A
turn-on energy by about 9%, 15%, and 22%, respectively, while removing the
larger `Lg` or `Ld` terms had little effect on turn-on energy. The lesson is
that sensitivity follows shared-path topology, not just inductance magnitude.

Their loop comparison also gives scale:

| Term | DPT fixture | Buck fixture |
| --- | ---: | ---: |
| PCB loop | 3.6 nH | 1.6 nH |
| Input-cap package | 0.3 nH | 0.3 nH |
| HEMT package | 3.0 nH | 3.0 nH |
| Diode package | 1.8 nH | 1.8 nH |
| Current-shunt package | 2.0 nH | - |
| Total loop | 10.7 nH | 6.7 nH |

Even with an optimized PCB, the power packages dominated the remaining loop
inductance. This is directly relevant when a FastHenry board extraction appears
too large or too small: check whether package leads, shunts, and SPICE-model
internal leads have already been included.

## Ringing and Extraction Methods

onsemi AND9410 describes a pragmatic VRM extraction flow: measure ringing period
or frequency and infer loop inductance from the resonant capacitance,

```text
L ~= (T / (2 * pi))^2 / C
  ~= 1 / ((2 * pi * f)^2 * C)
```

where `T` is ringing period and `f` is ringing frequency. The method then
distributes loop inductance by measuring ringing voltage between points around
the power loop. The note demonstrates modern low-voltage VRM values: about
1.51 nH total switching-loop inductance, with sub-nH segment contributions, and
10 to 20 nH gate-loop inductance if the gate loop is not carefully integrated.

This method is useful, but it is not a universal truth meter. Ring frequency
extracts the inductance that resonates with the effective capacitance in that
specific transient. For MOSFET switching nodes, the effective `Coss` is
voltage-dependent, nonlinear, and different depending on whether the relevant
quantity is energy, charge, or small-signal capacitance at the ring voltage. A
ring-derived `L` must therefore be checked against:

- the `Coss` chord or energy-equivalent capacitance over the observed voltage
  swing;
- package inductance already embedded in the vendor SPICE model;
- external leads intentionally added by the extraction deck;
- current shunts, probe grounds, and fixture leads;
- mutual coupling and polarity in shared source/gate paths;
- damping from ESR, channel conduction, snubbers, and lossy capacitance models.

## Design Implications

CSI is a switching-loss term, not just a ringing term. TI SLPA009 models the
high-side MOSFET CSI in a synchronous buck and shows why a larger CSI reduces
effective gate drive during hard switching. The app note's low-voltage
MOSFET/VRM package CSI priors are also useful as scale markers: stacked-die
Power Block/PowerStage packages can be below 100 pH, clip QFN around 450 pH,
wire-bond QFN around 850 pH, and SO8 above 1.5 nH. Those values are far below
the several-nH CSI measured in TO-220/TO-247-class through-hole devices, but
they should not be treated as high-voltage package constants.

Surface-mount and Kelvin-source packages are therefore not incremental
mechanical alternatives; they change the electrical control loop. A Kelvin
source or separate driver-source pin removes most package source inductance from
the gate loop even when the power loop still contains source-side inductance.

Dymond and Stark show that one through-hole package contribution is not
completely immutable. Their experiment reduced the dynamic insertion/loop
inductance of a TO-220 Schottky diode in a specific GaN-switched test circuit;
it did not demonstrate reduced MOSFET CSI or TO-247 behavior. A custom heatsink
and copper-tape arrangement placed conductive surfaces close to the diode's
internal current paths, using image-plane currents to oppose magnetic field
change. The oscillation period shortened from about 11.5 ns to 10.5 ns and the
inferred loop inductance fell by 17%. This is probably a niche mitigation
because it adds mechanical precision and only partially reduces the problem, but
it is a useful research lever for packages that cannot be replaced.

## Guidance for Future Research

Use package-aware extraction contracts. Board copper extraction should say
whether it is pad-to-pad, pin-to-pin, or die-plane-to-die-plane. SPICE model
metadata should say whether the vendor model already includes internal package
leads. Never add extracted package leads on top of a lead-inclusive model. The
canonical dcdc-tools contract is
[fet-package-boundary.md](fet-package-boundary.md).

Represent package inductance as a network when the topology needs it. At minimum
keep `Ld`, `Ls`, `Lg`, and mutuals or CSI flags separate. For cascodes,
multi-die modules, Kelvin-source devices, or clips, preserve internal shared
links rather than fitting one scalar loop inductance.

Prefer measured package priors over geometry guesses when the mold hides the
bond wires. Aikawa-style CSI measurement is attractive because it can work on an
assembled PCB and directly measures the shared source path, provided the fixture
layout keeps injection/sense mutual inductance out of the result. Use
destructive inspection or X-ray only when building a package library.

Use ring tests as cross-checks, not as sole calibration. A switching-node ring
can validate the total effective loop seen by the transition, but it cannot by
itself allocate inductance between board copper, package leads, model-internal
leads, probes, and nonlinear capacitance error.

Treat heatsinks and nearby metal as electromagnetic parts at high `di/dt`.
Normally this is a risk for unmodeled eddy-current damping or coupling. Dymond
and Stark show it can also be exploited deliberately, but only with close
spacing and enough conductor thickness.
